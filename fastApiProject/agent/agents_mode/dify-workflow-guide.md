# Dify 工作流：从零到一玩转 AI 编排

> 一句话摘要：使用 Dify 可视化工作流，像搭积木一样快速构建生产级 AI 应用。

## 引言

你有没有遇到过这种情况？老板让你做个「智能客服机器人」，你算了算工期：接大模型要 3 天、知识库要 5 天、调试 API 又要 2 天……等上线，黄花菜都凉了。

传统 AI 开发就是这样，门槛高、周期长。但现在有了 Dify —— 一个开源的 LLM 应用开发平台，它把 AI 开发变成了「搭积木」：拖拽节点、连接线、配置参数，一个能跑的工作流分分钟搞定。

这篇文章会讲清楚三件事：**工作流是什么**、**怎么搭**、**有哪些坑要避开**。适合想快速上手 AI 应用开发的同学。

## 核心概念

### 什么是工作流？

你可以把 Dify 工作流理解成一条「AI 流水线」：原材料进去，产品出来，中间的加工步骤就是节点。

就像工厂流水线上的机械臂，每个节点只做一件事，但串起来就能完成复杂任务。Dify 提供了 20+ 种节点，包括：

| 节点类型 | 作用 | 类比 |
|---------|------|------|
| LLM | 调用大模型 | 流水线上的「大脑」 |
| 知识库检索 | 从文档中找答案 | 流水线的「资料库」 |
| 代码执行 | 运行 Python/JS | 流水线的「工具人」 |
| 条件判断 | 分支逻辑 | 流水线的「分岔路口」 |
| 迭代 | 循环处理 | 流水线的「重复工位」 |

### Workflow vs Chatflow

Dify 有两种应用类型，傻傻分不清会走弯路：

- **Workflow（工作流）**：单轮执行，适合翻译、总结等「一问一答」场景
- **Chatflow（对话流）**：多轮对话，有记忆能力，适合客服机器人

简单记：**需要上下文选 Chatflow，一次搞定选 Workflow**。

### 变量系统

Dify 的变量就像编程语言中的数据类型。搞懂这个，能少踩一半坑。

支持的类型：

```
String（字符串）
Number（数值）
Object（对象）
Boolean（布尔值）
Array [string]（字符串数组）
```

**重要原则**：节点连线只是指定执行顺序，不自动传变量！你必须手动在下游节点的输入中选择变量。

## 实战演示

### 场景：构建一个「文章摘要 + 关键词提取」工作流

这个工作流接收一篇文章，输出摘要和关键词。分四步：

### 第一步：创建工作流

1. 打开 Dify，点击「创建应用」
2. 选择「工作流」
3. 起个名字：「文章分析助手」

### 第二步：拖入节点

从左侧节点面板拖入以下节点：

```
开始 → LLM(摘要) → LLM(关键词) → 结束
```

### 第三步：配置开始节点

在「开始」节点添加输入参数：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| article | 段落文本 | 待分析的文章 |

### 第四步：配置 LLM 节点

**LLM(摘要) 节点配置：**

```yaml
模型：gpt-4o-mini
提示词：
"""
请为下面的文章生成一段简短摘要（不超过100字）：

{{article}}
"""
```

**LLM(关键词) 节点配置：**

```yaml
模型：gpt-4o-mini
提示词：
"""
从以下文章中提取3-5个关键词，用逗号分隔：

{{article}}

输出格式：关键词1, 关键词2, 关键词3
"""
```

### 第五步：配置结束节点

在「结束」节点添加输出：

| 输出名 | 来源变量 |
|-------|---------|
| summary | LLM(摘要) 的 text 输出 |
| keywords | LLM(关键词) 的 text 输出 |

### 第六步：测试运行

点击「试运行」，输入一篇文章，点击执行，查看结果。

### 进阶：添加条件判断

如果文章太短（< 100 字），直接返回原文，不需要摘要：

```
开始 → 条件判断 → [短文] → 直接返回原文
                  └→ [长文] → LLM摘要 → 结束
```

条件配置：

```
文章长度 < 100 → 短文分支
文章长度 >= 100 → 长文分支
```

## 常见问题 / 踩坑记录

### 问题 1：变量类型不匹配

**现象**：选了一个变量后，第二个变量找不到。

**原因**：Dify 变量有类型限制，类型不一致就无法聚合。

**解决**：使用「模板转换」节点强制转换为 string，或用「代码执行」节点构建特定类型。

```python
def main(article: str) -> dict:
    # 确保输出是字符串类型
    return {"result": article}
```

### 问题 2：多分支结果无法汇聚

**现象**：用 if-else 分支后，下游节点报「变量未定义」。

**原因**：条件分支只有一条路径执行，另一条路径的变量不存在。

**解决**：在分支汇合处添加「变量聚合」节点，将多路输出合并。

### 问题 3：迭代节点输出顺序错乱

**现象**：用迭代处理数组，结果顺序不对。

**原因**：迭代开启并行执行后，输出顺序是完成顺序而非输入顺序。

**解决**：如果需要保持顺序，关闭并行执行选项。

### 问题 4：API 调用失败

**现象**：发布为 API 后调用报错。

**解决**：

1. 检查 `/parameters` 接口返回的入参格式
2. 文件上传需要用 `multipart/form-data`
3. 流式输出需要使用 `text/event-stream`

## 总结

### 核心要点

- Dify 工作流通过「节点 + 连线」实现 AI 流程编排，无需编码基础
- Workflow 适合单轮任务，Chatflow 支持多轮对话
- 变量必须手动传递，连线只控制执行顺序
- 代码执行节点是万能工具，可用于类型转换、API 调用、数组构建
- 条件分支后用变量聚合器汇合结果
